Versionshinweise

Auf dieser Seite können die Änderungen und Fehlerbehebungen der jeweiligen Version eingesehen werden.

Version 1.1.20220506

Feature:
  • Pro Parkplatz kann von der Besitzerin/dem Besitzer nun ein/e Vertreter/in (mehrere möglich) benannt werden.

Version 1.1.20220201

Fehlerbehebung:
  • Die Übersicht der eigenen Parkplätze hat in manchen Fällen nicht alle Parkplätze korrekt angezeigt. Der Fehler wurde behoben, gleichzeitig wurde ein Indikator hinzugefügt, ob der jeweilige Parkplatz aktiv ist oder nicht.

Version 1.1.20220126

Feature:
  • Die Parkplatzbörse ist jetzt fähig verschiedene Parkplatzpools zu verwalten. Daher muss nun bei der Registrierung von neuen Userinnen und Usern der jeweilige Pool ausgewählt werden.
  • Die Django-Version wurde von 3.2.10 auf 4.0.1 upgedatet.

Version 1.0.20220104

Fehlerbehebungen:
  • Wenn "Regelmäßige Freigaben" erfasst wurden, gab es nach dem Speichern dieser kein Feedback. Jetzt wird man auf die Übersichtsseite der Parkplätze zurückgeleitet.
  • Die Farbgestaltung und Benennung von freigegebenen und gebuchten Parkplätzen wurde verbessert. Die Hilfeseite ist entsprechend angepasst worden.

Version 1.0.20220103

Feature:
  • Über den Funktion "Regelmäßige Freigaben" können nun gewisse Wochentage automatisiert freigegeben werden, ohne dass manuelle Buchungen notwendig sind. Dies ist pro Parkplatz konfigurierbar und kann unter "Meine Parkplätze" ausgewählt werden.

Version 1.0.20211214

Fehlerbehebung:
  • Wenn ein Nutzer/eine Nutzerin schon einen Parkplatz angelegt hatte, fehlte in der eigenen Parkplatzübersicht der Button zum Anlegen eines weiteren Parkplatzes. Dieser ist nun vorhanden.

Version 1.0.20211213

Feature:
  • Auch ein/e Parkplatzbesitzer/in kann nun einen fremden Parkplatz buchen, wenn dieser freigegeben wurde. Dafür muss der eigene Parkplatz dann aber freigegeben worden sein. Solange der andere Parkplatz von ihm/ihr gebucht ist, kann die Freigabe des eigenen Parkplatzes nicht aufgehoben werden. Ebenso ist in dieser Konstellation keine Zurückforderung möglich. Dafür muss die Reservierung des anderen Parkplatzes erst gelöscht werden.
  • Der Besitzer/die Besitzerin eines Parkplatzes wird nun auch namentlich in der Übersicht angezeigt.
  • Die Zahl der Datenbankzugriffe bei der Generierung der Übersicht wurde signifikant reduziert
Fehlerbehebung:
  • Das Passwort wurde nicht korrekt gespeichert. Der Fehler wurde korrigiert.
  • Nach einer Passwortänderung wurde auf die falsche Seite verlinkt.

Version 1.0.20211212

Dies ist die Startversion
